# Getting Started with the Fyra Backend

This guide will teach you how to use and extend the Fyra compiler's backend.

## Overview

The Fyra backend is a multi-target code generator that transforms Fyra IR into textual assembly or binary machine code. It supports multiple architectures including x86_64, AArch64, RISC-V 64, and WebAssembly.

## Using the Backend in Code

To generate code for a Fyra `Module`, you need to:
1. Select a `TargetInfo` implementation.
2. Initialize the `CodeGen` engine.
3. Call `emit()`.

```cpp
#include "codegen/CodeGen.h"
#include "codegen/target/SystemV_x64.h"

// ... parse or build your module ...

auto target = std::make_unique<codegen::target::SystemV_x64>();
codegen::CodeGen cg(*module, std::move(target), &std::cout);
cg.emit(); // Emits textual assembly to stdout
```

### Binary Output

To generate binary machine code instead of textual assembly, pass `nullptr` as the output stream and call `emit(true)` if you want to generate an entry point (like `_start`).

```cpp
codegen::CodeGen cg(*module, std::move(target), nullptr);
cg.emit(true);

// Get the generated bytes
const auto& code = cg.getAssembler().getCode();
```

## Supported Targets

| Target | Architecture | ABI | Format |
|--------|--------------|-----|--------|
| `SystemV_x64` | x86_64 | System V (Linux/macOS) | Text/ELF |
| `Windows_x64` | x86_64 | Windows | Text/PE (partial) |
| `AArch64` | ARM64 | AAPCS64 | Text/ELF |
| `RiscV64` | RISC-V 64 | LP64D | Text/ELF |
| `Wasm32` | WebAssembly | Wasm | Text/Binary |

## Extending the Backend

To add support for a new instruction or optimize an existing one, you typically modify the relevant `TargetInfo` subclass (e.g., `src/codegen/target/AArch64.cpp`).

### Example: Adding a new Scalar Instruction

1.  Locate the `emitOp` method in your target class.
2.  Implement the textual emission path (`if (auto* os = cg.getTextStream())`).
3.  Implement the binary emission path (`else { ... }`) using the `Assembler`.

### Memory Management

The backend provides a built-in bump allocator for the `alloc` instruction. This allocator uses two special symbols:
- `__heap_base`: The start of the 1MB heap area.
- `__heap_ptr`: A 64-bit pointer to the next free byte.

When implementing a new target, ensure `emitAlloc` correctly increments `__heap_ptr`.

## Optimization Passes

Before code generation, the module can be optimized using various passes:
- **SCCP**: Sparse Conditional Constant Propagation.
- **DCE**: Dead Code Elimination.
- **GVN**: Global Value Numbering.
- **Mem2Reg**: Promotes memory allocations to registers/stack slots.

Example of running an optimization:
```cpp
transforms::SCCP sccp(error_reporter);
sccp.run(*function);
```
